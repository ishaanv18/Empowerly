## ğŸ¯ Problem Statement

Modern organizations face numerous challenges in managing their workforce effectively:

### **1. Fragmented HR Systems**
- Multiple disconnected tools for attendance, leave, payroll, and performance management
- Data silos leading to inefficient workflows and poor decision-making
- High operational costs due to multiple software subscriptions

### **2. Manual & Time-Consuming Processes**
- Manual attendance tracking prone to errors and buddy punching
- Paper-based leave applications causing delays in approvals
- Time-consuming payroll calculations with high error rates
- Tedious performance review cycles lacking standardization

### **3. Lack of Employee Engagement**
- Limited channels for employee feedback and recognition
- Absence of personalized skill development recommendations
- Poor internal communication and collaboration tools
- Low employee morale due to lack of motivation initiatives

### **4. Security & Compliance Concerns**
- Inadequate access controls and audit trails
- Difficulty in monitoring suspicious activities
- Compliance challenges with data protection regulations
- Risk of unauthorized access to sensitive employee data

### **5. Limited Insights & Analytics**
- Lack of real-time dashboards for HR metrics
- Inability to identify patterns in attendance, leave, and performance
- No predictive analytics for workforce planning
- Difficulty in generating compliance reports

#### **Authentication Flow**
```

     â†’ User enters OTP â†’ Backend verifies â†’ Create user â†’ Return JWT token
     â†’ Frontend stores token â†’ Subsequent requests include JWT in headers

```

         â†’ Store in MongoDB â†’ Return success

## ğŸš€ Installation

### **Prerequisites**
- **Java 17** or higher
- **Node.js 18** or higher
- **MongoDB Atlas** account (or local MongoDB)
- **SendGrid** account for email
- **Google Cloud** account for Gemini API
- **Git** for version control

### **Backend Setup**

1. **Clone the repository**
   ```bash
   git clone https://github.com/ishaanv18/Empowerly.git
   cd Empowerly/empowerly-backend
   ```

2. **Configure environment variables**
   
   Create environment variables in your IDE or export them:
   ```bash
   export MONGO_URI="mongodb+srv://username:password@cluster.mongodb.net/empowerly"
   export JWT_SECRET="your-256-bit-secret-key-here"
   export SENDGRID_API_KEY="SG.your-sendgrid-api-key"
   export GEMINI_API_KEY="AIzaSy-your-gemini-api-key"
   export ADMIN_EMAIL="your-admin-email@example.com"
   ```

3. **Build and run**
   ```bash
   mvn clean install
   mvn spring-boot:run
   ```

   The backend will start on `http://localhost:8080`

### **Frontend Setup**

1. **Navigate to frontend directory**
   ```bash
   cd ../empowerly-frontend
   ```

2. **Install dependencies**
   ```bash
   npm install
   ```

3. **Configure API URL** (Optional for local development)
   
   Create `.env` file:
   ```env
   VITE_API_URL=http://localhost:8080/api
   ```

4. **Start development server**
   ```bash
   npm run dev
   ```

   The frontend will start on `http://localhost:5173`

### **Access the Application**
- Open your browser and navigate to `http://localhost:5173`
- Register a new account (you'll receive an OTP via email)
- Verify OTP and login

   - Configure:
     - **Root Directory**: `empowerly-backend`
     - **Runtime**: Docker
     - **Instance Type**: Free (or as needed)

   - Go to [Vercel](https://vercel.com)
   - Click **Add New** â†’ **Project**

     - **Framework Preset**: Vite
     - **Root Directory**: `empowerly-frontend`

3. **Deploy**
   - Click **Deploy**
   - Vercel will build and deploy your frontend
   - Your app will be live at `https://your-app.vercel.app`

## ğŸ”’ Security Features

### **Authentication & Authorization**
- **JWT-based authentication** with secure token generation
- **BCrypt password hashing** for secure password storage
- **OTP verification** for email validation during registration
- **Role-based access control** (EMPLOYEE, HR, ADMIN)
- **Endpoint-level security** with Spring Security

### **Session Management**
- **Session tracking** with IP address, device, and location
- **Auto-logout** after 30 minutes of inactivity
- **Concurrent session detection**
- **Session invalidation** on logout

### **Security Monitoring**
- **Login attempt logging** (successful and failed)
- **Unusual activity detection** (unusual times, locations, multiple failures)
- **Real-time security alerts** for suspicious activities
- **Comprehensive audit trails**

### **Data Protection**
- **Environment variables** for sensitive configuration
- **HTTPS/TLS** for encrypted communication
- **CORS configuration** to prevent unauthorized access
- **MongoDB encryption at rest**
- **Sensitive data masking** in logs

### **Best Practices**
- **Input validation** on all endpoints
- **SQL injection prevention** (NoSQL, but still validated)
- **XSS protection** with React's built-in escaping
- **CSRF protection** with Spring Security
- **Rate limiting** (can be added with Spring Cloud Gateway)
